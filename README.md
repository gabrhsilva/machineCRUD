# 🚀 Machine CRUD

## 📖 Contexto

Um sistema centralizado permite visibilidade sobre todos os equipamentos em operação, seus tipos, setores e histórico.

Este projeto faz parte do desafio de onboarding da MIFY, com o objetivo de conhecer as tecnologias e o fluxo de trabalho da empresa através de um projeto prático de escopo reduzido.

---

# ✅ Requisitos Concluídos

## 🔧 Back-end

- ✅ Implementação sem auxílio de IA
- ✅ Criação de rotas REST para máquinas:
  - ✅ GET
  - ✅ POST
  - ✅ PUT
  - ✅ DELETE
- ✅ Validação de dados
- ✅ Persistência no PostgreSQL utilizando Prisma
- ✅ Respostas estruturadas em JSON
- ✅ Uso correto de status HTTP
- ✅ Modelo de máquina contendo:
  - ✅ Nome
  - ✅ Tipo
  - ✅ Setor
- ✅ Campos obrigatórios validados

---

## 🎨 Front-end

- ✅ Desenvolvimento realizado com auxílio de IA (Claude)
- ✅ Tabela com listagem de máquinas
- ✅ Exibição das colunas:
  - ✅ Nome
  - ✅ Tipo
  - ✅ Setor
- ✅ Modal/Dialog para criação de máquinas
- ✅ Validação de formulário
- ✅ Edição de máquinas existentes
- ✅ Exclusão de máquinas com confirmação
- ✅ Utilização de componentes MUI:
  - ✅ Table
  - ✅ Dialog
  - ✅ TextField
  - ✅ Button
- ✅ Tratamento de estados de carregamento
- ✅ Tratamento de erros na interface

---

## 🔗 Integração

- ✅ Integração entre Front-end e Back-end
- ✅ Consumo correto das APIs
- ✅ Exibição de mensagens de sucesso
- ✅ Exibição de mensagens de erro

---

# 🛠️ Tecnologias Utilizadas

## Back-end
- Node.js
- Express
- TypeScript
- Prisma ORM
- PostgreSQL

## Front-end
- React
- TypeScript
- Material UI (MUI)

---

# 📌 Objetivo

Desenvolver uma aplicação CRUD completa para gerenciamento de máquinas, aplicando boas práticas de desenvolvimento, arquitetura e integração entre serviços.
